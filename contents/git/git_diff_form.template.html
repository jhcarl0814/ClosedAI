what to diff:

<commit_parent>…

<path>... Are used to limit the diff to the named paths (you can give directory names and get diff for all files under them).

<pathspec>... If both paths point to directories, additional pathspecs may be provided. These will limit the files included in the difference. All such pathspecs must be relative as they apply to both sides of the diff.

how to diff:

diffcore-break: for splitting up complete rewrites

diffcore-rename: for detecting renames and copies

diffcore-merge-broken: for putting complete rewrites back together

diffcore-pickaxe: for detecting addition/deletion of specified string

-S<string> Look for differences that change the number of occurrences of the specified <string> (i.e. addition/deletion) in a file.

when -S<string>:

-G<regex> Look for differences whose patch text contains added/removed lines that match <regex>. Unless --text is supplied patches of binary files without a textconv filter will be ignored.

--find-object=<object-id> Look for differences that change the number of occurrences of the specified object.

diffcore-order: for sorting the output based on filenames

diffcore-rotate: for changing at which path output starts

whitespace errors:

--ignore-matching-lines=<regex> Ignore changes whose all lines match <regex>.
